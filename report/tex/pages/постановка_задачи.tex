\section{Постановка задачи}

Необходимо смоделировать поток пользовательских заданий на основе статистики реальной очереди SLURM, подготовить учебный кластер для запуска этой нагрузки и проанализировать полученные результаты выполнения.

Задачи.

\begin{itemize}
    \item Создать пять виртуальных машин: один управляющий узел и четыре вычислительных узла.
    \item Настроить статическую адресацию, разрешение имён и сетевую связность между всеми узлами.
    \item Настроить SSH-доступ по ключам между узлами без ввода пароля.
    \item Установить и настроить MUNGE для единой аутентификации.
    \item Установить SLURM, сформировать конфигурационный файл через официальный конфигуратор и разложить его на все узлы.
    \item Проверить состояние вычислительных узлов средствами SLURM.
    \item Отфильтровать исходный набор данных по индивидуальному варианту.
    \item Рассчитать параметры распределений фактического времени выполнения и ошибки прогноза времени.
    \item Реализовать генератор пакетных заданий SLURM.
    \item Запустить сгенерированные задания на кластере и собрать статистику выполнения.
    \item Представить результаты выполнения и сравнить исходные, сгенерированные и фактические характеристики заданий.
\end{itemize}

Во варианте работы обозначены: дистрибутив openSUSE Leap, UID 50728, имя задания \mintinline{text}{qe7}. Исходные записи набора данных фильтруются по указанному UID и имени задания.

Результатом работы должны стать настроенный кластер SLURM, исходный код генератора, набор сгенерированных заданий, результаты выполнения на кластере, сравнение расчётных и фактических характеристик и отчёт, оформленный в соответствии с ГОСТ 7.32--2017.
